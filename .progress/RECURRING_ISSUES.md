# 🔁 跨 Sprint 循环问题追踪 (Recurring Issues Tracker)

> **用途**：Phase R 审查开始前必须读取本文件，审查结束后必须更新本文件。
> 防止同类问题在多轮 Phase R 间反复出现却从未被根治。
> 
> **规则**：
> - 如果上次 Phase R 发现的问题本次依然存在 → `recurrence_count += 1`，升级优先级
> - 如果问题已被修复 → 标记 `✅ resolved`，记录修复版本
> - `recurrence_count >= 3` 的问题自动升级为 🔴 P0，且触发 escalation（暂停等用户确认）
